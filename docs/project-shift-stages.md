# Project SHIFT ステージ仕様

## 構成

- ステージは `src/features/project-shift/stages/` に1面1 JSONで配置する。
- ファイル名は `t01-01.json` から `t10-10.json`。
- Tierは10段階、各Tierは10面、通し番号は1〜100。
- 最大盤面サイズは `18x12`。
- `scripts/generate-project-shift-stages.mjs` が100 JSONを型付きデータへ統合する。
- `stage-data.generated.ts` は生成物のため直接編集しない。

## JSONスキーマ

```json
{
  "id": "t03-01",
  "tier": 3,
  "number": 21,
  "name": "圧力回路 導入",
  "briefing": "ステージ説明",
  "difficulty": 3,
  "mechanics": ["push", "switch"],
  "map": ["########", "#P.CG.E#", "########"],
  "legend": {
    "S": { "kind": "switch", "channel": "alpha", "goal": true },
    "D": { "kind": "door", "channel": "alpha" }
  },
  "solution": "RRDL"
}
```

`width`、`height`、プレイヤー位置、箱位置、タイル、PARは生成時に算出する。PARは既知解答の有効手数とする。

## マップ記号

| 記号 | 内容 |
| --- | --- |
| `#` | 壁 |
| `.` | 床 |
| `P` | プレイヤー |
| `C` | エネルギーキューブ |
| `G` | ゴールノード |
| `E` | 出口 |
| `I` | 氷床 |
| `^ > v <` | 一方通行 |

スイッチ、ドア、ワープは任意の1文字を `legend` に登録する。ドアには同じチャンネルのスイッチが必要。ワープはチャンネルごとに2地点必要。

`goal: true` のスイッチは圧力スイッチと最終配置先を兼ねる。箱数と通常ゴール・兼用ゴールの合計は必ず一致させる。

## 解答記法

- `U`: 上
- `D`: 下
- `L`: 左
- `R`: 右

解答には無効入力を含めない。`npm run test:game` は全100面の解答をエンジンで再生し、全入力が有効で、PARと手数が一致し、最終状態がクリアになることを検査する。

## Tier設計

| Tier | 主題 |
| --- | --- |
| 1 | 箱押し、回り込み、行き止まり |
| 2 | 複数箱、順序、デッドロック回避 |
| 3 | 圧力スイッチと単一ドア |
| 4 | 連動スイッチ・ドア |
| 5 | 一方通行と不可逆経路 |
| 6 | ワープによる経路分離 |
| 7 | 氷床と停止位置 |
| 8 | 複数ギミックの接続 |
| 9 | 3〜4種類の複合問題 |
| 10 | 全ギミックの総合試験 |

各Tierは1〜2面を導入、3〜5面を応用、6〜8面を複合、9面を高難度、10面を総合試験として扱う。

## 追加・編集手順

1. 対象JSONのマップ、説明、ギミック、解答を編集する。
2. `npm run generate:game-stages` を実行する。
3. `npm run test:game` で既知解答を検証する。
4. `npm run build` と `npm run check:site` を実行する。

ID、ファイル名、Tier内番号、通し番号をずらさない。未知記号、矩形でない盤面、箱と配置先の不一致、対応先のないドア、3個以上または1個だけのワープは公開前チェックで失敗する。
