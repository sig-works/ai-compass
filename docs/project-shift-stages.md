# Project SHIFT 倉庫番ステージ仕様

## ゲームルール

- プレイヤーは上下左右へ1マスずつ移動する。斜め移動はできない。
- 壁にはプレイヤーも箱も進入できない。
- 箱は1個ずつ1マス押せるが、引くことや複数箱をまとめて押すことはできない。
- 箱の移動先が空いた床または指定位置の場合だけ押せる。
- 指定位置へ置いた箱も、必要なら再び押して移動できる。
- すべての箱がいずれかの指定位置へ置かれた瞬間にクリアする。
- 箱と指定位置は同数とし、特定の組み合わせには紐付けない。

## データ構成

- `src/features/project-shift/stages/`へ1面1 JSONで配置する。
- 公開ステージは30面、3 Tier × 10面。
- ファイル名は`t01-01.json`から`t03-10.json`。
- `scripts/generate-project-shift-stages.mjs`がJSONを`stage-data.generated.ts`へ統合する。
- 生成物の`stage-data.generated.ts`は直接編集しない。

## JSONファイルの作成方法

現在は30面固定です。通常は既存JSONを複製して内容を編集し、同じ番号のステージを差し替えます。31面目以降を追加する場合は、ステージ検証、生成処理、UIのTier構成も変更する必要があります。

### ファイル名と番号

| ステージ番号 | Tier | ファイル名・ID |
| --- | --- | --- |
| 1〜10 | 1 | `t01-01`〜`t01-10` |
| 11〜20 | 2 | `t02-01`〜`t02-10` |
| 21〜30 | 3 | `t03-01`〜`t03-10` |

ファイル名と`id`は一致させます。`number`は全体を通した1〜30の連番です。

### 作成テンプレート

```json
{
  "id": "t01-01",
  "tier": 1,
  "number": 1,
  "name": "最初の一箱",
  "difficulty": 1,
  "map": ["#####", "#@$.#", "#####"],
  "boxCount": 1,
  "minMoves": null,
  "minPushes": null,
  "concept": "箱を押して指定位置へ置く",
  "solution": "R",
  "isValidated": true
}
```

各項目の意味は次のとおりです。

| 項目 | 内容 |
| --- | --- |
| `id` | `t01-01`〜`t03-10`形式の一意なID |
| `tier` | 難易度グループ。1〜3 |
| `number` | 公開順。1〜30 |
| `name` | ステージ選択画面に表示する固有名 |
| `difficulty` | 目安の難易度。1〜10の整数 |
| `map` | 1行を1文字列として記述したASCIIマップ |
| `boxCount` | 箱の総数。`$`と`*`の合計と一致させる |
| `minMoves` | 最短移動手数を確認済みの場合のみ整数。それ以外は`null` |
| `minPushes` | 最短箱押し回数を確認済みの場合のみ整数。それ以外は`null` |
| `concept` | その面で考えさせる内容を示す短い説明 |
| `solution` | `U`、`D`、`L`、`R`を連結した検証済み解答 |
| `isValidated` | 解答再生でクリアを確認済みなら`true` |

### マップの作り方

1. 幅3〜18マス、高さ3〜12マスの矩形を決める。
2. 全行を同じ文字数にする。
3. 上下左右の外周をすべて壁`#`で閉じる。
4. プレイヤーを`@`または`+`で1人だけ置く。
5. 箱を`$`または`*`で1個以上置く。
6. 箱と同数の指定位置を`.`、`*`、`+`で置く。
7. 箱を指定位置ではない角へ初期配置しない。
8. プレイヤーが箱の後ろへ回り込める経路と、箱を押すための空間を確認する。
9. すべての非壁マスが、箱を無視した状態でプレイヤー位置から到達可能になるようにする。

JSONでは行末の半角スペースが見えにくいため、床を行末へ置かず、必ず右端を壁`#`で閉じてください。タブ、全角スペース、ASCII表にない記号は使用できません。

## ASCII記号

| 記号 | 内容 |
| --- | --- |
| `#` | 壁 |
| 半角スペース | 床 |
| `.` | 指定位置 |
| `$` | 箱 |
| `@` | プレイヤー |
| `*` | 指定位置上の箱 |
| `+` | 指定位置上のプレイヤー |

`U`、`D`、`L`、`R`を解答の上下左右として使用する。`minMoves`と`minPushes`は最短性を確認できた場合だけ設定し、それ以外は`null`にする。

## 解答の作成方法

`solution`にはプレイヤーの全移動を開始位置から順番に記録します。箱を押した方向だけではなく、箱の後ろへ回り込む通常移動も必要です。

| 記号 | 移動 |
| --- | --- |
| `U` | 上 |
| `D` | 下 |
| `L` | 左 |
| `R` | 右 |

例として`"DDRURULLLL"`は、下、下、右、上、右、上、左、左、左、左の順に操作します。

- 無効な壁移動や、押せない箱への入力を含めない。
- 最後の入力で全箱が指定位置へ置かれることを確認する。
- 解答を確認するまでは`isValidated`を`false`にする。
- `isValidated: true`へ変更する前に`npm run test:game`を実行する。
- `minMoves`は解答文字数ではなく、最短解であることを確認できた場合だけ設定する。
- `minPushes`も最小の箱押し回数を確認できた場合だけ設定する。

## ステージ作成用プロンプト

生成AIへステージ案を依頼する場合は、次のプロンプトを基準にします。生成結果は必ず人間の確認と解答再生テストを通してください。

```text
正統派の倉庫番ステージを1面、JSON形式で設計してください。
使用できる要素は壁、床、指定位置、箱、プレイヤーのみです。特殊ギミックは使用しないでください。

制約:
- マップは幅3〜18、高さ3〜12の矩形
- 外周はすべて壁「#」で閉じる
- プレイヤーは「@」または「+」でちょうど1人
- 箱は「$」または「*」で1個以上
- 箱と指定位置の数を一致させる
- 許可記号は「#」「半角スペース」「.」「$」「@」「*」「+」のみ
- 指定位置ではない角へ箱を初期配置しない
- 箱を引かず、1個ずつ押す通常の倉庫番ルールで必ず解ける
- 一本道に押すだけではなく、箱の後ろへの回り込みや押す順序を含める
- solutionは開始位置からクリアまでの全移動をUDLRで記載する
- 最短性を証明できない場合、minMovesとminPushesはnullにする
- 説明文では「ゴール」や「ターゲット」ではなく「指定位置」と表記する

出力項目:
id, tier, number, name, difficulty, map, boxCount, minMoves, minPushes, concept, solution, isValidated

JSON以外の文章は出力しないでください。
```

## バリデーション

公開前チェックでは次を必須検証する。

- プレイヤーが1人、箱が1個以上存在する。
- 箱数、指定位置数、`boxCount`が一致する。
- マップが矩形で、外周が壁で閉じ、許可記号だけを使用する。
- 床領域がプレイヤーの静的到達範囲から分断されていない。
- `isValidated: true`のステージに解答があり、通常エンジンで全入力とクリアを再生できる。

指定位置ではない角の箱、指定位置へ運べない壁沿いの箱、箱を含む固定2×2領域は開発警告として出力する。警告はゲームの読込を停止しない。

## 難易度方針

- 1〜3面: 1箱で基本操作、回り込み、角の危険を確認する。
- 4〜11面: 2箱へ移行し、奥から置く順番、通路、一時退避を扱う。
- 12〜20面: 3箱、壁際、指定位置からの再移動、押し手の確保を扱う。
- 21〜30面: 3箱の依存関係、複数回の退避、不可逆な押下を組み合わせる。

ステージは完成状態から箱を逆方向へ動かして初期配置を作り、通常方向のソルバーで再度解答を確認する。難易度順は箱数、必要押下数、解答移動数を基準に整える。

## 追加・編集手順

1. `src/features/project-shift/stages/`内の対象JSONを複製または編集する。
2. ファイル名、`id`、`tier`、`number`の対応を確認する。
3. マップを作り、箱数、指定位置数、`boxCount`を一致させる。
4. 実際に操作して`solution`を記録する。
5. `npm run generate:game-stages`を実行する。
6. `npm run test:game`で全解答を再生する。
7. 警告が出た場合は、意図した難所か単純なデッドロックかを確認する。
8. `npm run build`と`npm run check:site`を実行する。
9. PCとスマートフォンで盤面サイズ、説明、操作性を確認する。
