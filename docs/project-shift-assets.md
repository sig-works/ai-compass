# Game 画像アセット仕様

Gameの盤面は、壁・床・指定位置・箱・プレイヤーだけで構成します。

## SVGアセット

盤面要素は`public/project-shift/assets/`配下のSVGを使用します。すべて`viewBox="0 0 64 64"`です。

| パス | 用途 |
| --- | --- |
| `tiles/floor.svg` | 通常の床 |
| `tiles/wall.svg` | 壁 |
| `tiles/target.svg` | 箱を置く指定位置 |
| `entities/box.svg` | 通常の箱 |
| `entities/box-on-target.svg` | 指定位置に置かれた箱 |

操作ボタンや見出しには`lucide-react`のSVGアイコンを使用します。UI専用のPNG/WebPファイルは使用しません。

### SVG制作仕様

- ルート要素を`<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 64 64">`にする。
- 幅と高さは`viewBox`で管理し、固定の`width`と`height`は必須としない。
- 64×64の表示でも、壁・床・指定位置・箱を瞬時に区別できる形にする。
- 主要部分を外周から3px以上内側に収め、隣のマスへはみ出さないようにする。
- 線幅は原則1.5〜3pxとし、細すぎる装飾を避ける。
- ライト・ダーク背景の両方で輪郭が判別できるコントラストを確保する。
- 箱と`box-on-target`は同じ形状を基本とし、色・発光・マーカーで状態を区別する。
- 外部画像、外部フォント、JavaScript、`foreignObject`を含めない。
- ラスター画像をSVG内へBase64埋め込みしない。
- Illustratorなどの編集情報や不要なメタデータは書き出し時に削除する。
- ファイル名と配置先は変更しない。

外部ツールで作成したSVGも、上記条件を満たして同名ファイルへ上書きすれば利用できます。

### SVG作成用プロンプト

画像生成AIやデザイン支援ツールには、まず次の共通指示を与えます。

```text
正統派の倉庫番ゲームで使用する盤面アセットを、編集可能なベクターSVGとして作成してください。
近未来的ですが過度に装飾せず、視認性とゲーム状態の判別を最優先にしてください。
64×64ピクセルの正方形グリッド向け、viewBoxは「0 0 64 64」、主要部分は外周から3px以上内側へ配置してください。
フラットなベクター形状、明瞭な輪郭、1.5〜3pxの線幅、少ない色数で構成してください。
ライト・ダーク両方の背景で判別できるコントラストにしてください。
グラデーションは最小限とし、文字、外部フォント、外部画像、埋め込み画像、JavaScript、foreignObjectは使用しないでください。
出力はSVGコードのみとしてください。
```

要素ごとに、共通指示へ以下を追加します。

#### 床

```text
通常床を作成してください。プレイヤーや箱より目立たない淡い床面で、薄いグリッドまたは中央マーカーを入れてください。
指定位置と誤認する円形マークや、壁に見える太い枠は避けてください。
```

#### 壁

```text
進入できない壁を作成してください。床より明確に濃く、構造材またはパネルのような堅牢な形にしてください。
上下左右に並べても連続した壁として見え、箱とは異なるシルエットにしてください。
```

#### 指定位置

```text
箱を置く指定位置を作成してください。床を基調に、中央へ明瞭なリング、照準、または四方向マーカーを配置してください。
箱が重なっても意味が伝わるよう、外周にも状態を示す色を残してください。
```

#### 箱

```text
押して移動する箱を作成してください。倉庫用コンテナを簡略化した、中央が詰まった正方形シルエットにしてください。
壁とは色と内部構造を変え、64×64表示でも移動物であることが分かるようにしてください。
```

#### 指定位置上の箱

```text
通常の箱と同じ形状を維持しながら、指定位置へ正しく置かれた状態を作成してください。
青緑や金色のアクセント、外周リング、中央マーカーなどを使い、通常箱との違いを色だけに依存せず表現してください。
```

## キャラクター

キャラクターだけは既存のPNG/WebP画像を維持します。

- `characters/player-astronaut`
- `characters/player-drone`
- `characters/player-explorer`
- `characters/player-geometric`

各キャラクターは512×512のPNGとWebPを用意します。

### キャラクター画像仕様

- 画像サイズは512×512pxとする。
- PNGとWebPを同じ内容・同じファイル名で用意する。
- 背景は完全透過にする。
- キャラクターは中央へ配置し、上下左右に約8〜12%の余白を確保する。
- 全身が収まる正面または斜め前向きの立ち姿を基本とする。
- 小さなマスでも頭部と胴体の向きが分かる、明瞭なシルエットにする。
- 盤面の壁や箱と同化しない明度・色相にする。
- 床へ接する薄い影は使用できるが、画像の端まで広がる影は避ける。
- 文字、ロゴ、UI、背景風景、枠線を含めない。
- アニメーションは使用しない。

PNGを原本とし、同じ見た目のWebPへ変換します。透過を維持し、過度な非可逆圧縮による輪郭のにじみを避けてください。

### キャラクター作成用プロンプト

```text
正統派の倉庫番パズルゲームで使用するプレイヤーキャラクターを1体作成してください。
近未来の倉庫作業員または探索者をテーマにし、親しみやすく、落ち着いたSFデザインにしてください。
512×512ピクセル、背景は完全透過、キャラクター単体、全身、正面または斜め前向きの立ち姿にしてください。
上下左右に10%程度の安全余白を取り、中央へ配置してください。
64×64程度へ縮小しても頭、胴体、手足の向きが分かる、太く明瞭なシルエットにしてください。
白、濃紺、青緑を基調とし、壁や箱と区別できるアクセント色を加えてください。
細かな模様や写実的な質感は抑え、ゲーム用アイコンとして読みやすい仕上がりにしてください。
文字、ロゴ、武器、背景、床、枠、UI、複数キャラクターは含めないでください。
出力形式は透過PNGにしてください。
```

既存4種類に合わせる場合は、末尾へ次のいずれかを追加します。

```text
小型宇宙服を着た人型キャラクターとして作成してください。
```

```text
浮遊する小型探索ドローンとして作成してください。人型にはしないでください。
```

```text
フード付きの軽装探索者として作成してください。
```

```text
円、四角、三角を組み合わせた幾何学的な人型アバターとして作成してください。
```

## 背景

ゲーム画面の背景はCSSのグラデーションで描画します。背景画像は使用しません。

## 差し替え手順

1. SVGまたはキャラクター画像を仕様に合わせて作成する。
2. 対象ファイルを同名で`public/project-shift/assets/`へ配置する。
3. キャラクターはPNGとWebPの両方を更新する。
4. `npm run check:site`を実行する。
5. `npm run build`を実行し、PCとスマートフォンの両方で視認性を確認する。

## 検証

`npm run check:site`では、盤面SVGの存在と`viewBox`、キャラクター画像の形式と寸法を検証します。

検証コマンドはデザインの視認性までは判定しません。床と指定位置、壁と箱、通常箱と指定位置上の箱が小さい表示でも区別できるか、目視でも確認してください。
